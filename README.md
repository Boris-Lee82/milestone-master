## 📋 项目简介

Milestone Master 是一个功能完善的单机里程碑管理系统，帮助项目经理高效管理多个项目的里程碑、待办事项、项目总览和周报。系统基于 Vue 3 + Element Plus 构建，使用 IndexedDB 进行本地数据存储，无需服务器即可运行。

### ✨ 核心特性

- 🎯 **里程碑管理** - 管理商业决策点和技术决策点，支持版本管理
- 📝 **待办事项** - 完整的任务管理系统，支持多维度筛选和进度跟踪
- 📊 **项目总览** - 项目统计概览和健康度评估
- 📈 **预测与周报** - 自动生成周报和风险预警
- 🎨 **主题系统** - 支持默认、暗色、蓝色三种主题
- 💾 **数据管理** - 数据导出、导入和备份功能

## 🚀 快速开始

### 环境要求

- Node.js >= 14.0.0
- npm >= 6.0.0

### 安装步骤

```bash
# 克隆项目
git clone <repository-url>

# 进入项目目录
cd "Milestone Master"

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

访问 http://localhost:3000 即可使用系统。

### 构建生产版本

```bash
# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

## 📊 项目健康度评估

系统采用智能算法评估项目健康度：

- **基础分**: 100 分
- **逾期任务**: 每个扣 10 分
- **高优先级未完成任务**: 每个扣 5 分

**评分等级**:
- 🟢 80-100 分：良好
- 🟡 60-79 分：一般
- 🔴 0-59 分：风险

## 📖 文档

### 数据存储

- **存储方式**: IndexedDB + LocalStorage
- **数据库名**: MilestoneMasterDB
- **特点**: 数据持久化存储在浏览器中，单机系统无需服务器

### 重要规则

1. **版本号规则**: 版本号必须递增（如 SP001, SP002, SP003）
2. **任务关联**: 一个任务可以关联到多个版本
3. **数据备份**: 建议定期导出数据作为备份
4. **日期格式**: 
   - 里程碑决策点: `2026/01/15`
   - 待办事项日期: `02/28`（省略年份）

### 常见问题

<summary>点击展开常见问题</summary>

**Q: 如何添加新的项目？**  
A: 在"配置管理"页面，点击"添加项目"按钮。

**Q: 如何将任务关联到版本？**  
A: 在"项目里程碑"页面，点击版本后的"关联任务"按钮。

**Q: 版本号有什么限制？**  
A: 版本号必须递增，不能重复。

**Q: 数据存储在哪里？**  
A: 所有数据存储在浏览器的 IndexedDB 中。

**Q: 如何备份数据？**  
A: 在"数据管理"页面，点击"导出数据"按钮。

## 🛠️ 技术栈

- **前端框架**: Vue 3 (Composition API)
- **UI 组件库**: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router
- **构建工具**: Vite
- **数据存储**: IndexedDB + LocalStorage
- **日期处理**: Day.js
